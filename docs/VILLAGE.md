# VILLAGE

状态：Game 层村庄草案。村庄设计必须优先服务 `NORTH_STAR.md` 中“今天村子又发生了什么”的判断标准。

## 村庄定位

村庄不是背景图，也不是金币农场。村庄是长期成长系统，用来承接 Merge 的最终产物，并反过来改变玩家后续玩法。

## 核心原则

- 建筑默认一次性完成。
- 建筑完成后永久存在。
- 建筑提供新功能或新规则。
- 建筑不重复堆叠产出。
- 村庄复杂度可以逐步增加，但棋盘保持简单。

## 建筑状态

建筑状态：

1. 未发现：玩家还不知道。
2. 已显示：村庄中出现空地或轮廓。
3. 可建造：任务或物品已解锁。
4. 建造中：已交付部分需求。
5. 已完成：建筑功能生效。

MVP 可简化为：

- 未解锁。
- 可建造。
- 已完成。

## 建筑作用类型

### 解锁 Merge 分支

建筑完成后开启新的链路或延伸旧链。

示例：

- 鸡舍完成后，母鸡可以继续合成冠军鸡。
- 面包房完成后，面团可以延伸到乡村面包。

### 改变货车规则

建筑影响物流，但不造成无限产出。

示例：

- 仓库：货车容量 +5。
- 装卸台：每日首次拆货额外给 1 个相关物品。
- 村口招牌：解锁特殊配送事件。

### 提供自动化

自动化要有限制，避免替代玩家操作。

示例：

- 整理台：每 10 分钟可自动合并一个低级组合。
- 仓库：可暂存 2 个物品。
- 小推车：一键收集当前可交付最终物品。

### 解锁事件

建筑引入新的短期目标。

示例：

- 市场：出现村民需求。
- 游客中心：出现游客订单。
- 蜂场：触发季节性采蜜事件。

## MVP 建筑

### 鸡舍

需求链：

```text
鸡蛋 -> 小鸡 -> 母鸡 -> 鸡舍
```

完成效果：

- 村庄中出现鸡舍。
- 解锁母鸡后续链：冠军鸡、金鸡蛋。
- 后续可关联市场或节日事件。

设计目的：

- 教会玩家“最终物品离开棋盘进入村庄”。
- 展示村庄从空地变成建筑。

## 后续建筑池

以下内容不进入 MVP。

- 蜂场：解锁蜂蜜链和季节事件。
- 果园：解锁水果链。
- 磨坊：解锁小麦、面粉、面团链。
- 面包房：连接磨坊和市场。
- 市场：解锁村民需求和订单。
- 游客中心：解锁游客事件。
- 仓库：提供少量暂存空间。
- 装卸台：改善货车处理效率。
- 公告牌：显示每日村庄目标。

## 村庄视觉成长

村庄成长应可见：

- 空地变成建筑。
- 建筑周围出现小道、栅栏、作物或村民。
- 同一建筑不无限升级，但可有 1-2 个视觉状态。
- 新建筑让村庄布局越来越热闹。

视觉不要过度复杂，微信小游戏屏幕上应能一眼读懂。

## 建筑效果约束

允许：

- 解锁链路。
- 调整掉落表。
- 少量提升容量。
- 提供有限自动化。
- 触发事件。

谨慎：

- 提升补货速度。
- 增加额外奖励。
- 降低所有需求。

禁止：

- 无限金币产出。
- 无限等级叠加。
- 多建筑重复乘算。
- 让玩家必须挂机等待金币。
