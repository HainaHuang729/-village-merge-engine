# ECONOMY

状态：Game 层经济草案。所有数值都是待验证假设，不是实现要求；若与 `NORTH_STAR.md`、`RISKS.md` 或 `MVP.md` 冲突，以权威文档为准。

## 经济设计目标

Village Merge 不以无限金币产出为核心。经济系统服务于短局回归、整理压力和村庄成长。

目标：

- 玩家每天有货可处理。
- 建筑推进有明确节奏。
- 不产生指数级金币膨胀。
- 不让玩家被大量资源类型压垮。
- 建筑主要改变玩法，而不是产出更多货币。

## 核心资源

### 货物

货物是主要行动入口。

表现：

- 货车中的待处理货。
- 每隔一段时间补充。
- 有容量上限。

设计建议：

- 初始容量：20。
- MVP 目标容量：20-40。
- 补货间隔：2 分钟 1 件作为初始参数。
- 离线补货不超过容量上限。

### 棋盘空间

棋盘空间是实际限制资源。

它限制：

- 玩家能拆多少货。
- 能否继续 Merge。
- 是否需要整理。

棋盘空间比金币更重要。

### 建筑

建筑是永久成长资源。

建筑提供：

- 新 Merge 分支。
- 新掉落表。
- 新自动化。
- 新事件。
- 新 UI 功能。

建筑不应该默认提供大量金币。

### 金币

金币不是 MVP 核心。

如果引入，只能用于：

- 小额便利操作。
- 低级物品回收奖励。
- 建筑外观小装饰。

不能用于：

- 主要进度门槛。
- 无限升级。
- 替代 Merge 核心玩法。

### 时间

时间通过货车补货表达。

原则：

- 玩家离线会积累货物。
- 积累有限制。
- 不做无限挂机收益。

## 产出与消耗

主要产出：

- 货车补货。
- 拆货生成初级物品。
- Merge 生成高级物品。
- 交付最终物品生成建筑。

主要消耗：

- 拆货消耗货车货物。
- Merge 消耗两个低级物品。
- 交付消耗最终物品。
- 删除或回收消耗棋盘物品。

## 防经济崩溃规则

### 防无限产出

- 货车有容量上限。
- 补货只到上限。
- 建筑不直接无限生产货币。
- 自动化建筑有触发条件或冷却。

### 防指数成长

- 建筑不提供叠乘收益。
- 货车容量提升有硬上限。
- 补货速度提升只做少量阶段变化。
- 不做无限等级建筑。

### 防内容消耗过快

- 关键建筑需要多条链交汇时再开放。
- 新链分阶段解锁。
- 任务只引导一部分目标，不把所有内容一次性暴露。

### 防玩家卡死

- 可删除低级物品。
- 任务相关掉落略有保障。
- 棋盘满时禁止继续拆货。
- 关键链初级物品保持可获得。

## 节奏建议

MVP 初始参数：

- 货车容量：20。
- 补货间隔：120 秒。
- 棋盘：36 格。
- 初级链：1 条。
- 首个建筑完成时间：3-5 分钟。
- 第二个建筑完成时间：10-15 分钟。
- 第三个建筑完成时间：20-30 分钟。

这些参数只用于初始验证，后续需要基于实际手感调整。

## 付费与商业化

第一阶段不设计付费。

原因：

- 核心循环尚未验证。
- 过早商业化会污染经济设计。
- MVP 要先证明玩家愿意回来处理货车。

未来可评估：

- 额外货车批次。
- 限时整理工具。
- 装饰外观。
- 活动通行证。

所有付费项不得破坏棋盘整理和村庄成长的核心体验。
