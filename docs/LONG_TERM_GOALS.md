# LONG_TERM_GOALS

## 长期目标不是扩系统，而是扩村庄生命

Village Merge 一个月后的核心问题不是：

```text
还有多少物品链？
```

而是：

```text
这个村子是否像一个真的地方？
```

长期更新必须围绕村庄生活展开。

## 1 个月目标

玩家应该看到：

- 村庄从安静变得有声音。
- 有 3-5 个地点被恢复。
- 村民开始有可记住的日常。
- 货车成为村庄与外部世界的连接。
- 每天都有一件小事值得看。

不要急着做大量系统。

一个月内最重要的是证明：

```text
村庄变化可以驱动留存。
```

## 3 个月目标

村庄应该开始形成生活网络。

例子：

- 鸡舍和集市产生关系。
- 果园和游客产生关系。
- 老人、孩子、返乡青年之间有轻互动。
- 村口公告牌能预告未来事件。
- 特定建筑改变村里每天出现的人和行为。

此时可以逐步加入：

- 小集市。
- 游客。
- 返乡青年。
- 村民请求。
- 季节小事件。

但每个新增系统都必须让村庄更活，而不是增加消耗。

## 6 个月目标

Village Merge 应形成可持续内容结构：

- 建筑恢复线。
- 人物生活线。
- 季节活动线。
- 村庄事件池。
- 外部物流事件。
- 游客反馈线。

玩家长期回来的理由应当是：

```text
我想知道这个村下一阶段会变成什么样。
```

不是：

```text
我还差几百个材料。
```

## 12 个月目标

一年后，项目应该拥有自己的内容语言。

理想状态：

- 玩家能说出自己村子的变化历史。
- 每个建筑都有生活记忆。
- 每个季节都有不同村庄氛围。
- 玩家不是在追数值，而是在陪伴一个地方成长。
- 新内容更新像“村里又要办新事了”，而不是“新增 50 个关卡”。

## 长期内容方向

### 村庄日历

村庄可以有轻量日历：

- 集市日。
- 收获日。
- 游客日。
- 返乡青年聚会。
- 节气活动。

日历不应变成任务压力，而是生活节奏。

### 人物关系

人物关系要轻，不要做重剧情。

适合：

- 老人和孩子的日常。
- 返乡青年带来新想法。
- 游客留下反馈。
- 村民对新建筑的反应。

不适合：

- 长篇对白。
- 强制剧情关卡。
- 大量分支选择。

### 地点记忆

每个地点都应有“前后变化”。

例子：

- 空地。
- 清理中。
- 建成。
- 有人使用。
- 产生小事件。

地点要成为玩家的记忆点。

### 外部世界连接

玩家作为返乡大学生，外部世界很重要。

可以通过：

- 物流。
- 短视频。
- 互联网订单。
- 游客推荐。
- 城市同学来访。

但外部世界不能压过村庄本身。

## 长期更新禁区

不要用以下方式延长生命周期：

- 无限建筑等级。
- 大量重复订单。
- 高阶物品链无限膨胀。
- 日常任务清单堆叠。
- 强制上线收取离线收益。
- 复杂经济系统。

这些会让游戏变得更长，但不一定更好。

## 长期成功判断

成功不是只看玩家等级。

更应该看：

- 玩家是否记得村里的人和地点。
- 玩家是否期待某个村庄事件。
- 玩家是否截图分享变化。
- 玩家是否愿意说“我的村子”。
- 玩家是否愿意在几分钟里回来看一眼。
