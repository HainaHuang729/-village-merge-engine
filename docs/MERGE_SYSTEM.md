# MERGE_SYSTEM

状态：Village Merge 的 Game 层 Merge 规则草案。通用 Merge 引擎设计以 `MERGE.md` 和 `ENGINE_ARCHITECTURE.md` 为准。

## 目标

Merge 系统负责“整理”和“合成”的核心手感。第一版必须简单、稳定、可扩展。

## 棋盘

默认规格：6x6。

理由：

- 微信手机屏幕上可清晰点击和拖动。
- 有足够空间制造整理压力。
- 不会像大棋盘一样拖慢短局节奏。

棋盘规则：

- 每个格子最多放 1 个物品。
- 物品占用 1 个格子。
- 初版不做多格物品。
- 初版不做障碍格。
- 初版不做棋盘扩建。

## 物品

物品基础字段：

- itemId：物品类型。
- level：链内等级。
- chainId：所属 Merge 链。
- state：普通、锁定、待交付等。
- source：来自货车、任务、建筑或事件。

第一版物品只需要支持普通状态。

## Merge 规则

基础规则：

- 两个完全相同的物品可以合成。
- 合成后产生同链下一级物品。
- 被拖动物品移动到目标格。
- 目标格物品被消耗。
- 新物品放在目标格或拖动终点格，具体手感在 Prototype 验证。

不做：

- 三合一。
- 五合二。
- 随机暴击升级。
- 多元素配方。
- 复杂生产器。

## 拖动规则

基本交互：

1. 按下物品。
2. 物品浮起并跟随手指。
3. 可合并目标高亮。
4. 松手后判断目标。
5. 可合并则 Merge。
6. 空格则移动。
7. 不合法则回到原格。

移动规则：

- 拖到空格：移动。
- 拖到相同物品：合成。
- 拖到不同物品：初版不交换，回弹。

不交换的理由：

- 降低规则复杂度。
- 减少误操作。
- 让整理压力更明确。

后续可评估是否加入“拖到不同物品时交换”。

## 自动 Merge

MVP 默认不做全自动 Merge。

可选轻量辅助：

- 可合并目标高亮。
- 双击同类物品自动寻找最近可合并对象。
- 建筑解锁后提供局部自动整理。

原因：

- 过早自动 Merge 会削弱整理玩法。
- 自动化应由村庄建筑逐步带来，而不是默认免费能力。

## 最终物品

最终物品不是永久留在棋盘上的奖杯。

规则：

- 当物品达到某个建筑需求终点时，显示可交付状态。
- 玩家点击或拖到村庄入口进行交付。
- 交付后物品离开棋盘。
- 村庄建筑建成或功能解锁。

好处：

- 棋盘不会被最终建筑占满。
- 玩家感到产物真的进入村庄。
- 建筑成为长期成长记录。

## 链路升级

每条链有初始终点和后续延伸。

示例：

```text
鸡蛋
↓
小鸡
↓
母鸡
↓
鸡舍
```

鸡舍建成后：

```text
母鸡
↓
冠军鸡
↓
金鸡蛋
```

设计原则：

- 不重复建造同一鸡舍。
- 建筑建成后改变链路终点。
- 原有中级物品获得新用途。

## 掉落与棋盘压力

货车拆货默认生成低等级物品。

掉落应控制：

- 同链物品比例。
- 孤立物品数量。
- 当前任务相关物品概率。
- 玩家是否接近堵死。

初版可以使用固定权重，不做复杂动态算法。

## 防堵死规则

最小方案：

- 棋盘满时禁止拆货。
- 提供删除低级物品按钮。
- 删除需要二次确认或长按，避免误删。

后续建筑能力：

- 仓库：临时存放少量物品。
- 整理台：自动合并一个低级可合成组合。
- 回收站：把低级物品转为少量材料。

## 第一版 Merge 链

第一版只允许一条链：鸡舍线。

```text
鸡蛋 -> 小鸡 -> 母鸡 -> 鸡舍
```

MVP 不加入第二条链，也不加入鸡舍后的延伸链。先验证拆货、整理、Merge、鸡舍进入村庄的完整体验。

鸡舍后的“冠军鸡”和“金鸡蛋”属于 P2 扩展内容。
