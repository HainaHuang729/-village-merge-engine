# GAME_VISION

## 项目定位

Village Merge 是一款关于“返乡、整理、建设和重新点亮村庄”的微信小游戏。

玩家不是村长，不是农场主，也不是上帝视角的经营者。玩家是一名刚从城市回到家乡的大学生，带着物流、互联网和现代生活经验，帮助一个逐渐沉寂的小村重新恢复活力。

Merge 是工具。村庄变化才是结果。

## One Sentence Vision

玩家每天用十分钟处理今天送到村口的货物，把混乱整理成新的生活，让一个沉寂的小村每天都多一点人气和故事。

这句话是未来一年所有开发决策的最高标准。

## 游戏真正卖的是什么

不是金币增长。

不是订单完成。

不是收菜效率。

不是 Merge 图鉴。

真正卖的是：

```text
我今天回到村里，发现它又活过来了一点。
```

玩家离开游戏时应该记住：

- 鸡舍建好以后，村路边真的有鸡在走。
- 老人开始坐在门口喂鸡。
- 孩子追着小鸡跑。
- 村口有人讨论明天的集市。
- 昨天空着的地块今天有了生活痕迹。

如果玩家只记得“我多了 200 金币”，设计就失败了。

## 核心情绪

Village Merge 应该提供一种温和但明确的情绪：

```text
我没有统治这个村子。
我是在一点一点把它扶起来。
```

这和普通经营游戏不同。

经营游戏常让玩家优化系统。Village Merge 要让玩家感到自己在恢复关系、生活和地方记忆。

## North Star

项目的 North Star：

```text
玩家不是为了让数字越来越大而回来。
玩家是为了看看今天这个村子又发生了什么。
```

所有功能必须回答：

- 它是否让村庄变得更鲜活？
- 它是否创造了新的村庄事件、生活细节或人物互动？
- 它是否让玩家更想明天再看看？

如果答案是否定的，功能应该删除或重做。

## 设计边界

Village Merge 可以有：

- 拆货。
- Merge。
- 建设。
- 村庄生活变化。
- 轻量故事。
- 返乡青年、物流、互联网、集市、游客。

Village Merge 不应该变成：

- 收菜游戏。
- 订单刷单游戏。
- 装修换皮游戏。
- 金币挂机游戏。
- 无限合成图鉴游戏。

## 商业判断

这个项目值得做的前提是：

```text
村庄变化本身能成为留存理由。
```

如果最后依赖签到、体力、限时礼包、订单压力和数值膨胀来留住玩家，那它就没有自己的核心竞争力。

因此，早期 MVP 不应急着做大量系统，而应验证一个问题：

```text
玩家是否愿意为了看村子下一次变化而回来？
```
