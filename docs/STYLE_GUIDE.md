# STYLE_GUIDE

本文件定义 Village Merge 的视觉风格。后续所有正式资源和 AI 批量生成资源必须遵循。

## 风格定位

参考方向：

- Travel Town 的高识别 Merge 图标。
- Merge Mansion 的清晰物品递进。
- 动物森友会的温暖乡村感。
- Unpacking 的整理治愈感。

目标感受：

- 温暖。
- 乡村。
- 高饱和。
- 圆润。
- 治愈。
- 卡通。
- 轻松。

## 画面原则

- 物品优先可读性。
- 轮廓必须清楚。
- 小屏幕下仍能辨认等级差异。
- 同系列资源必须统一角度、光照、描边和阴影。
- 不做复杂纹理。
- 不做写实材质。

## 色彩

推荐色彩方向：

- 鸡舍链：奶黄、浅橙、米白、暖红。
- 货车：柔和蓝。
- 纸箱：暖棕。
- 村庄：草绿、木色、暖土色。
- UI 强调：暖光黄、配送蓝、完成绿。

避免：

- 大面积灰暗色。
- 单一棕色主题。
- 过高对比导致刺眼。
- 不同系列之间饱和度差距过大。

## 光照

统一光照方向：左上方。

表现：

- 左上方略亮。
- 右下方轻阴影。
- 高光柔和，不使用强金属反光。

## 描边

统一规则：

- 深色描边。
- 圆润边角。
- 物品描边比内部细节更重要。
- 128x128 图标中描边应清晰但不压迫。

占位资源当前使用：

- 深蓝灰描边。
- 右下轻阴影。
- 中央英文标签。

正式资源替换后可以去掉英文标签，但必须保持同样的可读性。

## 阴影

阴影方向：右下。

阴影原则：

- 轻。
- 软。
- 不超出图标太多。
- 不变成黑色脏边。

## 物品风格

Merge 物品应像可拿起的小物件。

要求：

- 居中。
- 主体占画面 70%-80%。
- 等级越高轮廓越丰富。
- 同链之间有明确递进。
- 最终物品要比普通物品更有完成感。

鸡舍链递进：

```text
鸡蛋：简单、圆形、低复杂度
小鸡：更可爱、更亮
母鸡：轮廓更明确
鸡舍：建筑感、完成感
```

## 建筑风格

建筑要像小模型。

要求：

- 圆润比例。
- 清晰屋顶和入口。
- 不做真实透视复杂结构。
- 村庄中可直接识别。

第一版只有鸡舍，不扩展其他建筑风格。

## UI 风格

UI 图标必须：

- 触屏友好。
- 轮廓清楚。
- 不依赖细小文字。
- 和物品图标同一套描边、阴影和光照。

UI 不应像系统工具软件，应保持村庄和物流主题。

## 动画风格

第一版只定义 3 个动画接口：

### Merge 成功

感觉：

- 快速。
- 轻弹。
- 小闪光。

建议时长：200-350ms。

### 建筑飞入村庄

感觉：

- 最终物品从棋盘飞向村庄。
- 运动轨迹清楚。
- 不需要复杂粒子。

建议时长：500-800ms。

### 光点飞出

感觉：

- 少量暖色光点飞向 UI。
- 不能喧宾夺主。

建议时长：300-500ms。

## AI 批量生成要求

生成正式资源时，所有 prompt 必须包含：

- `cozy village merge game`
- `rounded cartoon style`
- `soft top-left light`
- `dark clean outline`
- `transparent background`
- `128x128`

同一系列必须一次性批量生成，避免风格漂移。
