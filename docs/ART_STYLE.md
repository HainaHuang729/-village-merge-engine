# ART_STYLE

状态：Game 层美术草案。正式资源方向以 `GAME_VISION.md`、`STYLE_GUIDE.md` 和 `ASSET_GUIDE.md` 的最新约束为准。

## 美术目标

Village Merge 的美术应传达：

- 乡村。
- 整洁。
- 温暖。
- 高反馈。
- 轻松解压。

不要做成沉重写实农场，也不要过度卡通到影响物品识别。

## 关键词

- 明亮自然光。
- 清爽乡村。
- 手感柔和。
- 物品轮廓清楚。
- 建筑像玩具模型一样可爱但不幼稚。
- 动画反馈短促有力。

## 视觉方向

推荐：

- 2D 俯视或轻等距视角。
- 物品使用清晰图标风格。
- 村庄建筑使用轻等距小模型风格。
- 色彩丰富但不过曝。

避免：

- 大面积单一绿色或棕色。
- 过度写实纹理。
- 复杂背景压过棋盘。
- 小屏幕上难以分辨的细节。

## 场景层级

### 货车

货车是每天回归的第一信号。

表现建议：

- 停在村口。
- 货箱数量或堆叠程度能表达剩余货量。
- 拆货时有打开箱子、弹出物品、货位减少的反馈。

### 棋盘

棋盘要干净，格子明确。

表现建议：

- 木箱、草地或仓库地板质感。
- 格子边界清晰但不刺眼。
- 可合并目标有柔和高亮。
- 拖动物品时浮起并带轻微阴影。

### 村庄

村庄承接长期成长。

表现建议：

- 空地、道路、建筑轮廓清楚。
- 新建筑落地时有短动画。
- 建筑完成后有小环境变化。

## 物品设计原则

- 同一链物品轮廓应递进明显。
- 不同链物品颜色和形状应明显区分。
- 低级物品可以简单，高级物品要更有奖励感。
- 最终物品需要有“可进入村庄”的仪式感。

示例：

```text
鸡蛋：圆形白色小图标
小鸡：黄色小动物形态
母鸡：红冠和翅膀更明显
鸡舍：小屋轮廓，带鸡标识
```

## 动画反馈

MVP 需要的基础反馈：

- 拆货弹出。
- 拖动浮起。
- Merge 闪光。
- 新物品弹性落位。
- 最终物品交付飞向村庄。
- 建筑出现。

动画原则：

- 快。
- 清楚。
- 不遮挡操作。
- 不拖慢连续拆货节奏。

## 音效方向

基础音效：

- 拆货：纸箱打开或木箱轻响。
- Merge：轻快合成音。
- 高级合成：更亮的提示音。
- 建筑完成：短促庆祝音。
- 不能操作：柔和阻止音。

音乐：

- 轻松乡村氛围。
- 循环不疲劳。
- 不抢交互音效。

## 资源生产原则

- 先用占位图验证手感。
- MVP 阶段不追求完整美术集。
- 物品和建筑命名必须与配置 id 对齐。
- 资源规格后续统一管理，避免散落。
