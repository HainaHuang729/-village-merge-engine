# DIFFERENTIATORS

## 核心差异化

Village Merge 最大的差异不是“乡村题材 Merge”。

真正差异应该是：

```text
Merge 的结果不是更多物品，而是一个逐渐恢复生活的村庄。
```

如果玩家只是在乡村皮肤下完成订单，那就是换皮。

## 与 Merge Mansion 的区别

Merge Mansion 的核心吸引力是：

- 神秘故事。
- 装修推进。
- 大量物品链。
- 长期任务结构。

Village Merge 不应该正面模仿。

我们的核心应是：

- 返乡身份。
- 物流和外部世界连接。
- 村庄日常生活恢复。
- 建筑建成后有人使用。
- 每天发生小事，而不是只推进装修进度。

最大区别：

```text
Merge Mansion 修的是房子。
Village Merge 恢复的是村庄生活。
```

## 与 Travel Town 的区别

Travel Town 的核心是：

- 生产器。
- 订单。
- 大量链路。
- 持续消耗。

Village Merge 如果大量做订单，会失去特色。

我们应该避免：

- 村民只是订单板。
- 每天只是交付指定物品。
- 生产器占据棋盘。
- 高级链路不断堆叠。

最大区别：

```text
Travel Town 让玩家满足订单。
Village Merge 让玩家看见村庄因为自己的行动产生生活。
```

## 与 Hay Day 的区别

Hay Day 是成熟经营经济。

Village Merge 不应追求：

- 完整生产链。
- 农场库存管理。
- 商品买卖循环。
- 社交交易。

我们的重点不是经营效率，而是村庄恢复。

最大区别：

```text
Hay Day 经营一个农场。
Village Merge 帮一个家乡重新有人气。
```

## 为什么不是经营游戏换皮

经营游戏通常围绕：

- 生产。
- 加工。
- 销售。
- 扩张。
- 效率。

Village Merge 应该围绕：

- 拆货。
- 整理。
- 建设。
- 村庄生活反馈。
- 关系和记忆。

判断标准：

如果一个建筑建成后，玩家首先关心“它产出什么”，我们偏经营。

如果一个建筑建成后，玩家首先关心“村里会发生什么”，我们才成立。

## 为什么不是放置游戏

放置游戏的快乐来自：

- 离线收益。
- 数字增长。
- 自动化扩张。
- 更快产出。

Village Merge 可以有离线货车补货，但不能卖离线收益。

玩家回来不是收割，而是处理今天的村庄变化。

判断标准：

如果玩家离线越久，主要奖励是更多资源，这是放置游戏。

如果玩家回来后，主要期待是村里有新事情，这是 Village Merge。

## 为什么这是微信小游戏

微信小游戏适合 Village Merge，因为：

- 每天 10 分钟轻体验适合碎片时间。
- 竖屏操作适合拆货、拖动、整理。
- 治愈乡村题材适合广泛人群。
- 村庄变化适合截图和轻分享。
- 不需要复杂操作和长时间会话。
- 可长期以小内容更新维持生活感。

但微信小游戏也有风险：

- 用户耐心短。
- 前几分钟必须建立情绪。
- 加载和操作必须轻。
- 不能依赖复杂剧情铺垫。

因此，第一天必须让玩家看到村庄变活，而不是只学系统。

## 为什么玩家会推荐给朋友

可推荐点不是：

- 福利多。
- 抽奖爽。
- 数值强。

而是：

- “这个村子真的会变。”
- “建完鸡舍后有人在喂鸡。”
- “每天进去看村里发生什么很舒服。”
- “像把一个荒村慢慢救回来。”

推荐需要可截图、可复述、可情绪化。

## 不可替代体验

不可替代体验是：

```text
通过每天短短几分钟的拆货和整理，把一批普通物品变成村庄里永久存在的生活痕迹。
```

这件事同时包含：

- 短局爽感。
- 整理治愈。
- 建设成就。
- 村庄生命感。
- 长期陪伴。

少掉任何一项，都容易被竞品替代。
